# MVP: моделирование потоков и устойчивости сети

Демонстрационный прототип для оценки идей: генерация сети, расчёт потоков, анализ отказов, каскады, риск и Monte Carlo. **Не является операционной моделью энергосистемы.**

## 1. Что это

Интерактивное веб-приложение — **8 режимов** (вершины куба «масштаб × представление × тип модели»). Пользователь генерирует сеть, смотрит потоки, отключает объекты, запускает N-1, каскад и Monte Carlo, получает screening-метрики и Pareto-подсказки.

Это **исследовательский MVP**, а не dashboard с готовыми KPI и не «цифровой двойник» под эксплуатацию.

## 4. Как запустить

```bash
pip install flask
python app.py
```

Откроется http://127.0.0.1:5001 — **вход:** [index.html](http://127.0.0.1:5001/) (куб или guided demo · [start.html](http://127.0.0.1:5001/start.html)).

Документация: [readme.html](http://127.0.0.1:5001/readme.html) · [roadmap.html](http://127.0.0.1:5001/roadmap.html) · [backend-demo.html](http://127.0.0.1:5001/backend-demo.html) (фаза 3).

Расчёты в браузере — `graph_core.js`. **Фаза 3:** отдельный расчётный API (pandapower DC) и batch War-Gaming вне браузера.

### API v2 (pandapower, фаза 3)

```bash
pip install -r api/requirements.txt
python -m uvicorn api.main:app --port 8000
```

- OpenAPI: http://127.0.0.1:8000/docs  
- UI: http://127.0.0.1:5001/api-demo.html  
- `POST /dc-run` — DC screening (до 20 buses в MVP)  
- `GET /calibration/status` — заглушка контура калибровки

### Batch War-Gaming (фаза 3)

```bash
node scripts/run_mc_batch.mjs 1000 42
```

Результаты: `data/batch/latest_summary.json` + CSV. Просмотр: http://127.0.0.1:5001/batch-results.html

### Docker (фаза 3)

```bash
cp .env.example .env   # задайте локальный пароль Postgres (файл .env не в git)
docker compose up --build
```

Web :5001 · API :8000 · PostgreSQL :5432 (заготовка под сценарии).

## 4a. Для самостоятельного просмотра (без автора)

**Время:** ~7 минут · **Логин:** не нужен.

1. Откройте **start.html** → кнопка **«Демо: волна ударов»** (или сразу `eng-large-map.html?demo=1`).
2. Дождитесь окончания автодемо (~2–3 мин): сеть 100 узлов → 100 сценариев поражения → худший сценарий → Pareto top 3 на карте.
3. Посмотрите **Pareto-кривую** в правой панели.
4. Прочитайте **roadmap** — что из вашего ТЗ уже есть, что — этап 4. Честное сравнение с radai-1984.dev.

**Что это НЕ обещает:** операционный клон ОЭС Украины, AC/pandapower, миллионы прогонов, агент 24/7, продукты A2/A3. Это **screening-MVP контура A1** — доказательство, что цепочка «сеть → поражение → слабые места → что укрепить» работает end-to-end.

## 5. Главный demo path (~1 мин)

1. На главной откройте **start.html** или **режим 8** (`eng-large-map.html?demo=1` для автодемо).
2. Нажмите **Построить на карте** (или дождитесь автопостроения) — пресет 100 узлов.
3. На карте: **кластеры** узлов и **топ-10 risk** линий (по умолчанию, без перегруза линиями).
4. Справа: таблица **Top-10 risky objects**.
5. **War-Gaming** (100 прогонов) или кнопка **«Демо: волна ударов»**. Дождитесь завершения.
6. В правой панели кликните **worst scenario** в таблице (при автодемо — уже выбран).
7. В **Pareto summary** и на **кривой** — **Protect top 1 / 3 / 5** — подсветка объектов на карте.

Подсказка дублируется на `home.html` и в левой панели режима 8.

В UI на всех экранах — дисклеймер **synthetic / not operational**.

## 7. Ограничения MVP

- Нет AC/Newton-Raphson в браузере; **pandapower DC** — только в API v2 (до 20 buses).
- Нет привязки к реальной SCADA/EMS, нет актуальных режимов сети.
- Monte Carlo — демо (сотни–тысячи прогонов), не промышленный объём.
- N-1 на 300/1000 узлах — выборочный scan (sampled), не полный перебор.
- Все рекомендации — **screening**, не инженерное заключение.

## 8. Что нужно для production

- Верифицированная модель сети (CIM/EMS export), актуальные параметры.
- AC/операционные ограничения, защиты, нормативы.
- Валидация на исторических авариях и экспертная приёмка.
- Масштабируемый расчётный backend, аудит, роли доступа.
- Отдельный контур «рекомендации → согласование → внедрение», не автопилот.
