Письмо заказчику — MVP screening-симулятор сети

Тема: Демо MVP контура A1 — просмотр в браузере и опции развёртывания на сервере

Добрый день,

Направляю демонстрационный MVP для оценки концепции War-Gaming / screening устойчивости сети (контур A1 — физическая безопасность).

Главный вопрос demo: какой минимальный набор объектов укрепить, чтобы сеть пережила сценарии поражения?

Важно сразу: все данные synthetic, модель not operational — это screening-прототип, а не операционный клон ОЭС и не инженерное заключение.

Ограничения среды «компьютер заказчика»

- Для показа: пресет 100 узлов и автодемо — стабильно на типичном ПК.

- Пресет 1000 узлов + тысячи прогонов может тормозить вкладку.

- Мобильный браузер — не рекомендуется для режима 8.

- Старая страница «Граф потоков» — Ctrl+Shift+R или инкогнито.

Итог части 1

В браузере полностью рабочий end-to-end сценарий:

сеть на карте → сценарии поражения (War-Gaming) → слабые места → Pareto «что укрепить» — без установки ПО.

Backend фазы 3 в браузерном контуре не участвует — только документация и пример batch-JSON.
