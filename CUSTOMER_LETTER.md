## Часть 1. Что реально работает в браузере (подробно)

### Условия

| Параметр | Значение |
|----------|----------|
| Установка | **Не нужна** — только ссылка в браузере |
| Браузер | Chrome или Edge (рекомендуется), актуальная версия |
| Интернет | Нужен для первой загрузки страницы и внешних библиотек (карта, UI) |
| Где считается | **На компьютере пользователя** — в браузере, файл `graph_core.js` |
| Логин / пароль | Не требуется |

Расчёты **не** идут на нашем сервере. Мощность ПК пользователя влияет на скорость (особенно при 300–1000 узлах и большом числе прогонов War-Gaming).

### Чего в браузере **нет** (важно)

| Не работает / не обещается | Почему |
|----------------------------|--------|
| **Живой API pandapower** | API — отдельный Python-сервис; на GitHub Pages сервер не запускается |
| **api-demo.html → POST /dc-run** | Страница откроется, но запрос к API **упадёт**, если локально не запущен `uvicorn` |
| **Новый batch War-Gaming** | Нужен Node.js на машине, где запускают скрипт — не в браузере |
| **PostgreSQL, Docker, калибровка** | Только при серверном развёртывании (часть 2) |
| Реальная топология Укрэнерго | Synthetic demo; NDA-топология — этап 4 |
| AC / Newton-Raphson на полной сети | Нет в браузере |
| Привязка к SCADA / EMS | Нет |
| Миллионы прогонов Monte Carlo | В браузере — сотни–тысячи; промышленный объём — сервер + этап 4 |
| Агент 24/7, NL-запросы | Нет (design stub в документации) |
| ML-корректор, автокалибровка | Нет |
| Продукты A2, A3 | Вне scope MVP |
| Инженерное заключение | Screening-demo only |

**backend-demo.html** на GitHub Pages — **инструкция** для серверного контура, а не кнопка «запустить backend».

### Ограничения среды «компьютер заказчика»

- Для показа: пресет **100 узлов** и автодемо — стабильно на типичном ПК.
- Пресет **1000 узлов** + тысячи прогонов может тормозить вкладку.
- Мобильный браузер — не рекомендуется для режима 8.
- Старая страница «Граф потоков» — **Ctrl+Shift+R** или инкогнито.

### Итог части 1

В браузере **полностью рабочий** end-to-end сценарий:  
**сеть на карте → сценарии поражения (War-Gaming) → слабые места → Pareto «что укрепить»** — без установки ПО.

Backend фазы 3 в браузерном контуре **не участвует** — только документация и пример batch-JSON.

## Часть 2. Развёртывание на сервере (кратко)

Если есть сервер (у исполнителя или у заказчика) — можно поднять **расширенный контур** поверх того же UI.

### Возможно на сервере сейчас

| Что | Как | Результат |
|-----|-----|-----------|
| Полный UI | `python app.py` или Docker web :5001 | Все 8 режимов по сети |
| API pandapower DC | `uvicorn api.main:app` или Docker api :8000 | `POST /dc-run`, до **20 buses** |
| Batch War-Gaming | `node scripts/run_mc_batch.mjs 1000 42` | CSV + JSON, до **50 000** прогонов |
| Всё в Docker | `cp .env.example .env` → `docker compose up --build` | web + api + postgres |
| Инструкции backend | backend-demo.html | API + batch |

### Невозможно / не входит в поставку

| Что | Статус |
|-----|--------|
| Запись прогонов в PostgreSQL | Контейнер есть, **код не пишет в БД** |
| Распределённые workers, 10⁶+ прогонов | Этап 4 |
| Единый расчёт: режим 8 → pandapower API | Два контура, **не связаны** |
| AC на полной сети, топология NDA | Этап 4 |
| Калибровка sim vs observed | Только stub API |
| Агент 24/7, SCADA | Этап 4+ |
| Боевой EMS | Не цель MVP |

### Минимальный план (без Docker)

1. `git clone` → `pip install flask` → `python app.py` (порт 5001)  
2. Опционально: `uvicorn api.main:app --host 0.0.0.0 --port 8000`  
3. Опционально: `node scripts/run_mc_batch.mjs 1000 42`
