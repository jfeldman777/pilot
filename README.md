# MVP: моделирование потоков и устойчивости сети

Демонстрационный прототип для оценки идей: генерация сети, расчёт потоков, анализ отказов, каскады, риск и Monte Carlo. **Не является операционной моделью энергосистемы.**

## 1. Что это

Интерактивное веб-приложение (режимы 1–6 из 8 вершин куба «масштаб × представление × тип модели»). Пользователь генерирует сеть, смотрит потоки, отключает объекты, запускает N-1, каскад и Monte Carlo, получает screening-метрики и Pareto-подсказки.

Это **исследовательский MVP**, а не dashboard с готовыми KPI и не «цифровой двойник» под эксплуатацию.

## 2. Что реализовано

| Возможность | Где |
|-------------|-----|
| Генерация synthetic-сети (схема + карта) | режимы 1–6 |
| Математический flow / баланс | 1–4 |
| DC power flow, θ, loading | 5, 6 |
| DC N-1 анализ | 5, 6 |
| Каскадная симуляция (trip по перегрузке) | 5, 6 |
| Engineering attributes (kV, типы, repair time) | 5, 6 |
| Risk score, Top-10 risky objects | 5, 6 |
| Monte Carlo demo + Top worst scenarios | 5, 6 |
| Pareto recommendations (protect top 1/3/5) | 5, 6 |
| Планы укрепления (бюджет, critical) | 1–6 |

**Рекомендуемый сценарий демо:** режим 6 — инженерная карта (`eng-map.html`).

## 3. Как запустить

```bash
python app.py
```

Откроется http://127.0.0.1:5001 — выбор режима на `home.html`.

Требования: Python 3, Flask (`pip install flask`). Расчёты выполняются в браузере (`graph_core.js`), отдельный backend для вычислений не нужен.

## 4. Как пройти демо (3–5 минут)

1. На главной откройте **режим 6** — «Малый · Карта · Инженерная».
2. **Построить на карте** — сеть на реальных/синтетических координатах СЕС.
3. Убедитесь, что **DC power flow** выбран — линии окрашены по загрузке, в панели метрики DC.
4. **Запустить Monte Carlo** (100 runs, cascade = yes). Дождитесь `100 / 100 runs · готово` (кратко может мелькнуть `Pareto analysis…`).
5. В правой панели откройте **Top-10 worst** — кликните худший сценарий.
6. На карте: initial outage, cascade-tripped, **timeline** каскада.
7. В **Pareto summary** кликните Protect top 1 / 3 / 5 — подсветка объектов для укрепления на карте.

Подсказка **Demo path** дублируется в левой панели режима 6.

## 5. Что synthetic / open data

| Данные | Источник |
|--------|----------|
| Координаты СЕС | открытый реестр / sample JSON / synthetic fallback |
| Топология связей | synthetic (граф по seed) |
| Электрические параметры (R/X, capacity, kV) | synthetic |
| Repair time, risk score, damage score | synthetic screening |
| Pareto / Monte Carlo | demo-метрики, не защита и не инвестпрограмма |

В UI на всех инженерных экранах — дисклеймер.

## 6. Ограничения MVP

- Нет AC/Newton-Raphson, нет pandapower.
- Нет привязки к реальной SCADA/EMS, нет актуальных режимов сети.
- Monte Carlo — демо (сотни–тысячи прогонов), не промышленный объём.
- Режимы 7–8 (большая инженерная сеть) не реализованы.
- Все рекомендации — **screening**, не инженерное заключение.

## 7. Что нужно для production

- Верифицированная модель сети (CIM/EMS export), актуальные параметры.
- AC/операционные ограничения, защиты, нормативы.
- Валидация на исторических авариях и экспертная приёмка.
- Масштабируемый расчётный backend, аудит, роли доступа.
- Отдельный контур «рекомендации → согласование → внедрение», не автопилот.

## 8. Следующие шаги (после демо)

1. Согласовать приоритет: карта vs схема, малый vs большой масштаб.
2. Определить источники реальных данных и границу synthetic.
3. Выбрать целевые сценарии (N-1, каскад, планирование) для углубления.
4. Оценить интеграцию с существующими системами заказчика.
